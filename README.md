# SkippyGames

Магазин игр (Steam / PlayStation / Xbox / Nintendo Switch) и внутриигровой
валюты на GitHub Pages, с автоматическим сбором данных из Steam.

## Структура

```
index.html                         — весь сайт (каталог + карточка игры + чат + лайтбокс)
games.json                         — данные о играх (генерируются автоматически)
backend/parser.py                  — сборщик данных Steam -> games.json
backend/custom_ids.txt             — appid, которые нужно добавить вручную
backend/skipped_ids.json           — appid, которые парсер проверил и пропустил (не игры) — создаётся автоматически
backend/requirements.txt           — зависимости Python
.github/workflows/update_games.yml — автозапуск парсера каждые 6 часов
```

Сайт — это одна страница (`index.html`), переключающая между собой каталог и
карточку игры на чистом JavaScript, без перезагрузки страницы. Ссылка на
конкретную игру выглядит как `index.html?id=1245620`.

## Как это работает

1. `backend/parser.py` запускается по расписанию через GitHub Actions —
   каждые 6 часов (можно запустить и вручную: вкладка Actions → "Update
   games.json" → Run workflow).
2. Каждый прогон работает **инкрементально**:
   - сначала обновляются цены/данные у самых "старых" карточек в каталоге
     (тех, что не обновлялись дольше 3 дней);
   - затем добавляются новые игры — сперва из `backend/custom_ids.txt` и
     топ-продаж Steam, потом из полного списка приложений Steam
     (`ISteamApps/GetAppList`), пока не будет достигнута цель в 10 000 игр
     или не закончится бюджет времени на прогон (55 минут).
   - appid, которые оказались не играми (DLC, саундтреки и т.д.),
     запоминаются в `backend/skipped_ids.json`, чтобы не тратить на них
     запросы повторно.
3. Экшен коммитит обновлённые `games.json` и `backend/skipped_ids.json`
   обратно в репозиторий.
4. `index.html` загружает `games.json` через `fetch()` и рендерит всё на
   чистом JS — бэкенд-сервер не нужен.

### Почему каталог растёт не мгновенно, а постепенно

У Steam Store API нет официального публичного ключа для больших объёмов
запросов, и он ограничивает частоту обращений с одного IP. Чтобы не
получить блокировку и уложиться в лимит GitHub Actions (6 часов на один
job), парсер добавляет **до 600 новых игр за один прогон** и обновляет
**до 1200 существующих карточек**. При прогоне каждые 6 часов (4 раза в
сутки) каталог реалистично дорастает до 10 000+ игр примерно за 1–2 недели,
дальше просто поддерживается в актуальном состоянии (обновление цен).
Если нужно быстрее — увеличьте `MAX_NEW_GAMES_PER_RUN` в `parser.py` и
частоту cron в `update_games.yml`, но учитывайте риск временной блокировки
по rate-limit со стороны Steam.

## Реализованные требования

- **Трейлеры**: на карточке игры проигрывается реальное видео (`<video>`)
  из Steam CDN (`trailer_video` в `games.json`). Если у игры нет трейлера в
  Steam, честно показывается сообщение об этом — не заглушка "иконка вместо
  видео".
- **Чат**: виджет в правом нижнем углу — это официальный
  `VK.Widgets.CommunityMessages`, подключённый к сообществу по числовому
  ID (см. настройку ниже). Он реально инициализируется и открывает диалог
  с сообществом ВКонтакте, а не имитацию переписки.
- **Лайтбокс скриншотов**: клик по скриншоту на странице игры открывает
  полноэкранный просмотрщик с навигацией стрелками/клавиатурой (←/→/Esc).
- **Допродажи (DLC/донат)**: если у игры в Steam есть DLC (в том числе
  паки внутриигровой валюты), они подгружаются парсером с ценой и
  отображаются в блоке "Дополнения и внутриигровая валюта".
- **Полные описания**: `description` в `games.json` — это очищенное от
  HTML-разметки, но НЕ обрезанное `detailed_description` из Steam.
- **Футер**: указан владелец «Андрей «ShadowSkip»» со ссылкой на
  Telegram https://t.me/SKIPPYManager.
- **Изображения**: обложки — `capsule_616x353.jpg`, hero-баннер на
  странице игры — `library_hero.jpg`, скриншоты — в разрешении
  `path_full` (обычно 1920×1080), а не миниатюры.
- **10 000+ игр**: см. раздел выше про инкрементальный рост каталога;
  на фронтенде реализована пагинация (24 игры на страницу) и debounce
  поиска (200 мс), чтобы UI не тормозил даже при большом каталоге.
- **Жанры и платформы**: полный список из 44 жанров и 4 платформ (PC,
  PlayStation, Xbox, Nintendo Switch) в виде фильтров в сайдбаре каталога.
  Учтите: сам Steam не сообщает о доступности игры на PlayStation/Xbox/
  Switch — эти платформы для конкретных игр нужно проставлять вручную
  (см. ниже), иначе все игры из Steam будут помечены только как "PC".

## Настройка перед запуском

### 1. Числовой ID группы ВКонтакте

В `index.html` в блоке `<script>` в самом начале:

```js
var VK_GROUP_NUMERIC_ID = 195484236;
```

Если понадобится сменить сообщество — узнать числовой ID можно в разделе
"Статистика" сообщества (в адресной строке будет `vk.com/stats?gid=ЧИСЛО`),
либо через https://vk.com/dev/CommunityMessages. Убедитесь, что в
настройках сообщества (Управление → Сообщения) включена галочка
"Разрешить использование виджета", и что домен вашего сайта на GitHub
Pages добавлен в поле "Разрешённые домены" там же.

### 2. Кастомный список игр и платформ

Добавляйте appid игр построчно в `backend/custom_ids.txt`. Если у игры
есть релизы на PlayStation/Xbox/Nintendo Switch, а не только на Steam —
после генерации `games.json` отредактируйте поле `"platforms"` у нужной
игры вручную (парсер сам знает только про Steam/PC).

### 3. Аватарки дежурного менеджера

В `index.html` массив `MANAGER_AVATARS` содержит 7 рабочих ссылок-заглушек
(placehold.co) — по одной на каждый день недели. Замените `url` на
реальные изображения персонажей, когда будут готовы.

## Деплой на GitHub Pages

1. Загрузите все файлы проекта в репозиторий (ветка `main`).
2. Settings → Pages → Source → ветка `main`, папка `/ (root)`.
3. Settings → Actions → General → Workflow permissions → "Read and write
   permissions" (нужно, чтобы Action мог коммитить `games.json`).
4. Запустите workflow вручную первый раз: Actions → "Update games.json" →
   Run workflow.
5. Откройте `https://<ваш-логин>.github.io/<репозиторий>/`.

## Локальный запуск парсера

```bash
cd backend
pip install -r requirements.txt
python parser.py
```

Каждый локальный запуск дособирает каталог точно так же, как и в GitHub
Actions — если запускать его локально несколько раз подряд, каталог тоже
будет постепенно расти до 10 000 игр.
